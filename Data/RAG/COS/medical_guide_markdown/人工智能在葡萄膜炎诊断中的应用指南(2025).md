## 1 《人工智能在葡萄膜炎诊断中的应用指南（2025）》制定背景和方法

葡萄膜炎是一种常见的威胁视力的眼内炎症性疾病，占全球视力受损原因的 $5 \% \sim 1 0 \% ^ { \mathrm { ~ [ 1 ] ~ } }$ ］。视力丧失最常见的原因是黄斑水肿、青光眼和视网膜缺血［2］ 。2021年，葡萄膜炎命名标准化（SUN）系统根据解剖位置、发病情况、持续时间、病程和严重程度制定了葡萄膜炎的分类标准［3］ 。葡萄膜炎病因复杂，表现形式多样，与大约 80种不同的病因有关，其中最常见的是感染性、自身免疫性或自身炎症性疾病，鉴别诊断比较困难［4］ ，这可能导致过多的检测和费用。此外，眼部特征（如眼部病变部位、慢性病和相关体征）和眼外特征（临床检查和全身检查）中存在大量潜在相关变量，使得病因诊断变得复杂［4-5］ 。在欧美国家，约四分之一的葡萄膜炎诊断与单独的眼科疾病有关，约四分之一与确诊的全身性疾病有关，约四分之一与疑似全身性疾病有关， $23 \% \sim 4 4 \%$ 的患者病因仍未确定［6］ 。对于缺乏经验或非专业的临床医师而言，葡萄膜炎是一种难以诊断和分类的疾病［7］ 。近年来，人工智能（AI）在海量数据的分析、解释和利用方面显示出越来越大的实用性，已被提议作为辅助疾病鉴别诊断的工具［8-9］ 。在眼科领域中，AI算法已应用于干眼［10］ 、白内障［11］ 、青光眼［12］和眼底疾病［13］ ，这些算法大多数也可以应用于葡萄膜炎眼科检查。最近，研究人员发现 AI技术在葡萄膜炎的诊断、分类和确定潜在病因方面表现出潜力［14-15］ 。AI算法有助于处理辅助检查和影像学中的重要发现，从而有助于诊断葡萄膜炎。

由于葡萄膜炎诊断的复杂性以及缺乏强有力的证据或共识来指导诊断方法，中国医药教育协会智能医学专委会和国际转化医学会眼科学专委会于 2023年 6月成立《人工智能在葡萄膜炎诊断中的应用指南（2025）》专家组，组织执笔专家、葡萄膜炎专家、眼科影像专家于 2023年 7月 3日认真学习国内外 AI在葡萄膜炎中的应用研究文献，并结合眼科 AI临床研究的实践经验，召开线下和线上会议，针对 AI在葡萄膜炎中的应用进行充分讨论和论证，并分析当前面临的挑战以及未来发展方向。由执笔专家组成员撰写指南初稿，初稿形成后通过电子邮件和微信方式由各位专家独立阅读并提出修改意见，分别提交指南撰写组核心成员，修改意见经过整理后，通过微信、邮件等方式，并借助线上会议平台进行讨论，随后加以归纳，指南在修改期间充分接受参与专家的建议和指导意见，最终达成指南终稿，旨在帮助眼科医生理解、研究和临床应用 AI技术。本指南制定过程历时 1年余。

## 2 葡萄膜炎中的 A

葡萄膜炎最常用的分类由国际葡萄膜炎研究小组于 1987年制定，其基于解剖学受累的位置将葡萄膜炎分为前葡萄膜炎、中葡萄膜炎、后葡萄膜炎和全葡萄膜炎等四种类型［16］ 。2005年，SUN小组对此进行了扩展［17］ 。根据病原体不同，分为感染性（细菌、病毒、真菌或寄生虫）葡萄膜炎、非感染性或伪装性葡萄膜炎［16］ 。感染性葡萄膜炎在发展中国家更为普遍，最常见的病因是弓形虫病和结核病。其他主要的感染性病因包括梅毒、盘尾丝虫病、囊虫病、麻风病和钩端螺旋体病［18］ ，在发达国家，非感染性或免疫介导性葡萄膜炎更为常见。

AI在葡萄膜炎中的应用，主要体现在潜在病因诊断、分类和评估方面［19］ 。在眼科领域，临床数据与影像学检查（如眼底照相和 OCT）相结合，可用于诊断、管理和跟踪眼部疾病。因此，专家组构建了 A辅助图像分析诊断葡萄膜炎疾病系统流程图，推荐大家参考使用（图 1）。在葡萄膜炎的筛查中，主要应用了计算机视觉模型，通过眼前节图像（例如裂隙灯、眼前段照相）或者眼底视网膜图像（例如眼底照相、OCT等）数据集来训练筛查模型，利用预处理技术和优化器来实现更好的性能，预处理包括调整大小和增强过程（随机裁剪、旋转、亮度变化和翻转）以及多个神经网络架构。在葡萄膜炎的检测中，主要通过应用检测软件和卷积神经网络来检测不同部位，例如眼前段和玻璃体炎症体征，并对前、后 OCT和眼底照片上的炎症病变进行分级。检测和分级眼前段和玻璃体炎症对于葡萄膜炎至关重要。然而，在一些发现检测算法中，评估是通过 OCT进行的，而OCT并不是进行此类评估的理想检查。计算机视觉卷积神经网络也存在一些技术限制，例如仅基于对图像的分析进行诊断以及需要大量数据集，算法在新数据中的表现较差，尤其是在小数据集上训练的算法。在葡萄膜炎的诊断中，主要采用了机器学习（ML）模型，包括贝叶斯信念网络（BN）、多层基于规则专家系统（ES）以及分类和回归树（CART），在病因匹配、数据集和疾病分布方面取得了良好的效果。丰富的眼科临床和影像学数据是一种宝贵的资源，可以帮助开发 AI模型。

## 3 AI在葡萄膜炎中的应用

### 3.1　诊断支持算法

### 3.1.1　眼前节炎症的量化

目前量化前房角（AC）炎症的金标准是使用裂隙灯显微镜进行临床检查来对细胞进行主观计数和闪辉分级［17］ 。AI已应用于医学成像，以协助诊断葡萄膜炎。研究证明，与使用裂隙灯进行的临床检查相比，基于 AI的前节光学相干断层扫描（AS-OCT）图像分析与 SUN分类在量化眼前节炎症方面具有显著且独立的相关性［20］ 。研究证明，通过软件可对OCT图像中 AC细胞存在的高反射点手动或自动化进行计数评估，并可与 AC细胞的临床分级相关联。与手动方法相比，自动化方法更敏感，检测能力更强［21］ 。同样，Sorkhabi等［22］开发了一种基于 AI的自动化方法来量化眼前节中的炎症，结果显示，临床SUN分 级 与 AI软 件 检 测 到 的 炎 症 细 胞 计 数（Spearman $\rho = 0 . 7 0 7 \ 7$ ）和炎症细胞密度（Spearman$\rho = 0 . 7 0 3 \ 5 \ \mathrm { ~ }$ ）之间均存在显著的相关性（均为 $P <$ 0．001）。该研究表明，基于 AI的自动化 AS-OCT图像分析可对 AC炎症进行无创和定量评估，在前葡萄膜炎的早期检测和管理中具有明显的应用价值。Ozer等［23］ 获取 21例 Fuchs葡萄膜炎综合征（FUS）患者的 SD-OCT眼前节图像，并上传到 ImageJ程序进行进一步分析，结果发现，FUS患者中角膜内皮细胞密度降低可能导致白内障手术后出现严重并发症，并且在 FUS患者中，颞侧虹膜色素上皮色素脱失发生在鼻侧之前。

### 3.1.2　玻璃体炎症的量化

与前房炎症类似，目前对玻璃体炎症进行评分的黄金标准是使用间接检眼镜进行临床检查，然而所有这些“临床”方法也受到屈光介质透明度的影响，并且无法检测到玻璃体炎症的细微变化。研究证明，与裂隙灯的临床检查结果相比，基于 AI的 ASOCT图像分析可以通过测量玻璃体提供可靠性和准确性更高的结果［24］ 。Keane等［25］从 30例葡萄膜炎患者的 OCT图像集中获得玻璃体信号强度的测量值，并导入到经过验证的定制分级软件中，然后将其与视网膜色素上皮（RPE）的光密度进行比较，生成具有任意单位的光密度比（“VIT／RPE-相对强度”），结果发现，VIT／RPE-相对强度与临床玻璃体混浊评分呈显著正相关 $r = 0 . 5 6 6$ 6， $P = 0 . 0 0 0 \ 1$ ），这项研究结果表明，OCT衍生的玻璃体信号强度测量可能有助于作为葡萄膜炎患者的结局测量。此外，Terheyden等［26］ 收集 49例葡萄膜炎患者以黄斑为中心的

OCT体积扫描，在固定位置自动分析每个体积扫描的 9个 B扫描，将以下 B扫描选择与每次体积扫描9次 B扫描的平均得分进行比较，作为参考标准：13／5／7次中心扫描 $\left( 1 \mathrm { c } / 3 \mathrm { c } / 5 \mathrm { c } / 7 \mathrm { c } \right)$ ）和3次广泛分布的扫描（3w），结果发现，扫描选择 7c和 $3 \mathrm { w }$ 显示了接收器工作曲线下面积的两个最高区域（分别为0．985和0．965），研究表明，葡萄膜炎中玻璃体强度的自动定量是可靠的，并且只需要很少的 OCTB扫描。

### 3.1.3　脉络膜炎症的量化

脉络膜厚度已被认为是某些类型的葡萄膜炎的重要炎症生物标志物［27］ 。然而，大多数涉及葡萄膜炎脉络膜厚度的计算都采用手动分割［28］ 。除了脉络膜厚度外，还有测量脉络膜水平的流量衰减和血管分布［29］ 。Agrawal等［30］ 使用脉络膜血管来量化脉络膜炎症，将脉络膜血管指数（CVI）定义为增强深度成像 （EDI）-OCT中血管面积与脉络膜总面积的比值，通过使用 ImageJ处理后的自定义软件手动分割脉络膜实现，结果发现，在葡萄膜炎诊断和 3个月随访期间，CVI下降了 $6 . 2 \%$ ，显著高于对照眼的 CV$0 . 7 \%$ 。因此，CVI有可能成为监测后葡萄膜炎、全葡萄膜炎和脉络膜疾病临床病程的新工具［31］ 。

### 3.2　病因诊断算法

葡萄膜炎由多种病因引起，因此，关键步骤创建算法方法将有助于眼科医生得出病因学诊断。通过算法方法发现潜在病因可能是最有效的［32］ 。通常从病史采集和人口统计学（年龄、性别、种族和社会经济史）开始。其次，考虑眼部检查的特征（时间顺序、侧面性、解剖位置、角膜变化、眼压、肉芽肿性变化以及粘连、视网膜血管炎、视盘水肿、黄斑水肿以及局灶性或多灶性脉络膜炎和视网膜炎的存在）。第三，眼外疾病的体征可以指向潜在病因。鉴于系统方法的潜在优势，专家组推荐了几种常用的 AI算法来处理由此产生的临床数据流，包括 BN、ES、CART（表 1）。

### 3.2.1　BN

BN是概率图形模型。贝叶斯定理是由英国统提供了如何通过先验置信度和新证据来计算后验置信度的数学框架。该定理已被计算机科学家用于创建 AI相关的复杂网络，称为 BN，它的使用最近已扩展到社会科学、生物学和医学。BN在医学诊断中的应用已经很广泛，用于表示不确定的知识，计算条件概率，并可用于决策支持系统［14］ 。一个贝叶斯网络是一个有向无环图，由代表变量结点及连接这些结点的有向边构成。结点代表随机变量，结点间的有向边代表了结点间的互相关系（由父结点指向其子结点），用条件概率进行表达关系强度，没有父结点的用先验概率进行信息表达。它们的图形表示允许直观地解释所研究变量之间的因果关系，同时保留计算其参数估计所需的数学属性［14］ 。葡萄膜炎的特征取决于疾病的病因，在葡萄膜炎网络中，中心节点是葡萄膜炎的病因，它取决于年龄、性别和种族。根据给定的网络结构和完整数据来估计贝叶斯网络参数，进一步对葡萄膜炎最常见病因进行鉴别诊断和病因学诊断。

### 3.2.2 ES

ES是一种基于规则的推理机制的 AI系统。它通过将专家的知识和经验转化为一系列规则，实现了对专家级别的问题解决能力的模拟。ES的基本结构通常由长期记忆（LTM）、短期记忆（STM）、推理引擎以及可能用于提供解释的额外模块组成［33］ 。ES的 LTM包含领域知识，这些知识可以从人类专家、最终用户和文献中获取；ES的 STM模块包含最终用户提供的信息或系统在运行时获得的结论。系统结论是通过使用存储的知识处理 STM中的信息得出的。推理引擎模块是 ES的处理器，它使用 STM中的信息将知识工程师设计的逻辑规则应用于存储在 LTM中的知识，无论该知识是系统结论还是用户前因。ES可以提供一个功能来解释系统为什么会提出某些问题和／或系统如何得出结论。除了诊断目的之外，ES还可以设计为做出决策并提供支持和指导能力。医学领域中许多基于规则的 ES都是为诊断或决策支持而设计的。由此产生的 ES是一个基于规则的系统，可以帮助眼科医生做出最准确的诊断。正确及时的诊断可以防止严重并发症的发生。该系统指导普通眼科医生收集诊断所需的信息并生成鉴别诊断作为输出。ES可根据患者病史和临床眼科检查结果来诊断葡萄膜炎。该系统旨在帮助缓解葡萄膜炎专家短缺的问题，这在贫穷或发展中国家尤为严重，因为这种疾病在这些国家很常见，可能会导致严重的视力障碍。此外，ES还可以用作教育工具或支持决策。

### 3.2.3　CART

CART是在给定输入随机变量 $X$ 条件下输出随机变量 Y的条件概率分布的学习方法。CART假设决策树是二叉树，内部结点特征的取值为“是”和“否”，左分支是取值为“是”的分支，右分支是取值为“否”的分支。这样的决策树等价于对每个特征进行递归二分，进而把输入空间即特征空间划分成有限个单元，并在这些单元上确定预测的概率分布，也就是在输入给定的条件下输出的条件概率分布。它由树的生成、树的剪枝构成。CART分析基于递归分区分析方法。可以根据葡萄膜炎风险最高的变量将总组划分为亚组。再次对这些子组进行分区，直到无法进行进一步的显著分区时，生成具有最佳诊断准确性的树。诊断树中的终端节点用于对葡萄膜炎的风险进行分类，因此该算法能够实现高度可靠的诊断。该算法的性能可能会随着在疾病过程中收集的额外信息而提高，这些诊断概率将有助于临床医生决定在某些时间点进行更具体的治疗。

## 4 性能指标

### 4.1　骰子相似系数

在许多不同的精确度指标中，我们使用骰子相似系数（DC）来衡量 AI辅助诊断系统的精确度。此外，我们还让 $G$ 和 $S$ 分别表示在地面实况分割和机器分割中标记为玻璃体的像素集合。DC指标定义如下：

$$
\mathrm { D C } = { \frac { 2 \times \mathrm { T P } } { 2 \times \mathrm { T P } + \mathrm { F N } + \mathrm { F P } } }
$$

其中，TP是 $S$ 和 $G$ 的交集的卡入度， $\mathrm { F P } = \left| S - G \right|$ ｜，$\operatorname { F N } = | G - S |$ ｜。

### 4.2　豪斯多夫距离

豪斯多夫距离（HD）用于衡量玻璃体分割边界与地面实况对应边界的接近程度。从 $G$ 到 $S$ 的 HD定义为最大欧氏距离 $d ( g , s )$ ），即在 $G$ 中的点 $^ { g }$ 和它们在 $S$ 中的最近点 $s$ s之间的距离。在一般情况下，从$G$ 到 $S$ 的 HD与从 $S$ 到 $G$ 的 HD是不对称的。双向豪斯多夫距离（BHD）是 HD的一种扩展，其定义为$\mathrm { B H D } _ { G  S } = \operatorname* { m a x } \{ \mathrm { H D } _ { G  S } , \mathrm { H D } _ { S  G } \}$ ｝。HD的公式如下：

$$
{ \mathrm { H D } } _ { G  S } = \operatorname* { m a x } _ { g \in G } \{ \operatorname* { m a x } _ { s \in S } \} d ( g , s ) \} \}
$$

### 4.3　葡萄膜炎症分级准确率

葡萄膜炎可以根据受炎症影响的眼部区域更具体地分为四种类型［33］ ：（1）前葡萄膜炎，是指影响眼睛前房的炎症；（2）中间葡萄膜炎，炎症累及玻璃体；（3）后葡萄膜炎，炎症影响眼睛后部视网膜和脉络膜；（4）全葡萄膜炎，眼部所有主要部位均受累。玻璃体炎症的分级通常依赖于一些客观的、可量化的指标，如炎症细胞的数量、玻璃体的混浊程度、视网膜病变的严重性等。这些指标可以通过眼底检查、OCT、荧光素眼底血管造影等辅助检查手段来观察和记录。尽管存在客观标准，但不同观察者（即使是经验丰富的医师）可能会对这些标准有不同的主观判断。例如，对于玻璃体混浊程度的评估，不同医师可能会有不同的判断。此外，炎症活动的评估可能受到医师个人经验、技术水平和对疾病理解程度的影响。炎症通常有 6个等级（0、0．5、1、2、3、4级），对于炎症分级阶段评估，可以通过全连接神经网络（FCNN）作为分类器来评估玻璃体炎症的严重程度，计算了所有分类分级的平均准确率［35］ ，公式为：

$$
\mathrm { D C } \ = \ \sum _ { i = 1 } ^ { N } { \frac { 2 \times \mathrm { T P } _ { i } } { 2 \times \mathrm { T P } _ { i } + \mathrm { F N } _ { i } + \mathrm { F P } _ { i } } }
$$

## 5 AI在葡萄膜炎诊断中面临的挑战和未来发展方向

目前许多研究尚处于起步阶段，AI在眼科的应用面临着诸多挑战：（1）样本的规模和代表性对于算法训练和验证至关重要，缺乏大型且更具代表性的数据集是当前主要限制，阻碍了模型的公平性和普遍性。更多研究探索了其他葡萄膜炎病因、眼部参数，包括脉络膜厚度、视网膜和脉络膜血管，因此需要真实、具有生物标记功能、其他成像方式（如 OCTA）和更新的计算模型。在实际部署之前，共享数据和代码以及合作研究对于确保 AI和 ML开发的可重复性和公平性至关重要，模型的公平性和普遍性、标签质量、标准化、质量控制和同质化仍然是推动该领域发展的重要因素。（2）图像本质上也是丰富的数据库，因为理论上每个像素都可以被视为一个单独的输入参数［36］ 。这为揭示病理过程提供了机会，但也带来了挑战，特别是在组装注释良好的数据集时，这些数据集足够大，可以在 ML框架中训练分类算法时避免过度参数化，预测统计方法的进步可能及时减轻对如此广泛的输入数据的需求。在OCT中应用的一种有用方法是将细分方法与 AI驱动的疾病分类分离［37］ 。（3）疾病是一个连续体，即使对于训练有素、技术娴熟的人类专家来说，区分不同严重程度的阈值也是非常具有挑战性的。因此，高质量的地面真实标签对于模型性能的提升至关重要。鉴于监督学习的本质，这一点的合理性不言而喻。为了减轻这一限制的影响，未来研究应当增加对图像的采集数量，并邀请更多专家进行细致的图像标注。推动实践朝着疾病过程的设备无关表示的方向发展，将有助于实现不同研究人员间研究结果的可比性。此外，还需要更多对应于不同严重程度和临床评分的类别。因此，对于开发具有更好实际应用的模型，需要具有较多级别的数据集，生成对抗网络已经成为一种生成更多训练数据的强大方法。这种生成数据的替代方法可以解决大多数应用程序中训练样本数量有限的问题。虽然更大的训练数据集总体上可能会带来更好的分类性能，但使用迁移学习和中等规模的训练数据集也可以实现深度学习模型的高精度。

## 6 结束语

AI算法显示出良好的效果，可以帮助临床医生诊断、分类和确定葡萄膜炎的病因。未来的研究不仅应确定新技术的诊断准确性，还应确保它们与标准治疗相比能提高临床和健康经济结果。通过合作研究和数据共享纳入葡萄膜炎的其他病因和更全面的数据，对于开展精心设计、足够有力的 RCT也是至关重要的。现代 OCT设备整合了其自己的图像处理工具和输出指标的标准化，计算机视觉图像处理的使用对于开发客观且可量化的葡萄膜炎炎症评分至关重要，这些评分可以作为比当前 SUN评分更敏感的疾病活动指标，并为葡萄膜炎的临床试验提供更可靠的结果。此外，眼睛的多模态和定量成像有可能促进我们对动力学、细胞生物学、转录组学和蛋白质组学结构的理解，以及了解这种多因素过程是如何调节的。借助非侵入性技术我们有机会更精确和全面地区分不同状态。这些技术可用于对从转录组和微生物组的详细检查、多参数流式细胞术和蛋白质组学中收集的信息进行分层。AI在眼科领域具有巨大前景，可以为葡萄膜炎患者提供早期诊断和最佳治疗建议。
